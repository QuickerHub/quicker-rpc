## 卡点

### 1. multi-var-assign：number 变量 + 字面量赋值

**预期**：单步 evalexpression 设 `a=1,b=2,c=a+b`，showText 显示 `3`。

**实际**：Agent 写 `{a} = 1` → 预处理 `v_a = 1` →「指定的转换无效」（`varType: number` 时字面量 int 赋值失败）。

**修复（已 trace 验证）**：

```text
{a} = Convert.ToDouble(1);
{b} = Convert.ToDouble(2);
{c} = {a} + {b}
```

动作 `b63593ce-d494-40f9-a87e-18011c443d28` 已 patch；showText 显示 `3`。

**应对**：

- benchmark-prompt 与 draft skill 记录：`number` 变量字面量用 `Convert.ToDouble(n)` 或 RHS 为表达式。
- 文档 `expressions.md` 可补一行 number 陷阱（后续 Phase 4）。

### 2. clip-lines-expr：选型偏模块链

Agent 正确 get 各模块，但用 9 步而非单步 LINQ/`evalexpression`。功能可跑，benchmark B 轴扣分。

**应对**：pattern「clipboard-lines-normalize」记录「优先 expr，模块链为 fallback」。

### 3. headless MCP：autoReview

`local.autoReview: true` 会拒绝 qkrpc MCP。SDK `config.ts` 已设 `autoReview: false`。

### 4. 交互任务 F 轴
